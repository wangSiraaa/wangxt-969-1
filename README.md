# 养老上门服务工单系统 - Docker 快速启动指南

## 项目概述
================
这是一个完整的养老上门服务工单系统，包含需求提交、调度派单、护理员签到、家属确认、结算管理、异常处置等完整业务流程。

## 系统架构
=========
- 后端：Spring Boot 2.7 + JPA + H2
- 前端：Vue 3 + Element Plus + Vite
- 反向代理：Nginx

## 快速启动
==========

### 方式一：Docker Compose 一键启动（推荐）

```bash
cd /Users/mingyuan/workspace/sihuo/wangxtw3/969
docker-compose up -d --build
```

启动后访问：
- 前端页面：http://localhost
- 后端API：http://localhost:8080/api
- Swagger文档：http://localhost/swagger-ui.html
- H2数据库控制台：http://localhost/h2-console
  - JDBC URL: jdbc:h2:mem:eldercare
  - 用户名: sa
  - 密码: （空）

### 方式二：本地开发模式

#### 启动后端
```bash
cd backend
mvn spring-boot:run
```

#### 启动前端
```bash
cd frontend
npm install
npm run dev
```

前端开发地址：http://localhost:5173

## 派单校验规则
==============

派单时会同时校验以下规则：

1. **老人状态校验**：暂停服务期间不能派单
2. **护理员状态校验**：护理员需在岗状态
3. **资质校验**：护理员资质需满足护理等级+服务包要求
4. **服务区域校验**：护理员服务区域需包含老人所在区域
5. **风险等级校验**：高风险老人需对应高等级护理员
6. **时间冲突校验**：同一护理员同一时间不能有多个工单
7. **排班容量校验**：不能超过护理员当日最大单量
8. **连续工时校验**：不能超过最大连续工作时长
